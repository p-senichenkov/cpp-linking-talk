# Динамические библиотеки
- Всё это для PIC
- Global offset table -- содержит абсолютные адреса
  - Заполняет ld (в рантайме)
- Procedure linkage table
  - Сначала содержит push некого индекса функции и вызов резолвера
  - Для функций в PLT в GOT сначала лежит возврат в PLT (как раз на `push`)
  - Резолвер ищет символ в .so и записывает в GOT

# Storage duration and linkage
- Storage duration
  - Static
    - Переменная определена в namespace scope или помечена `static` или `extern`
    - Она не помечена `thread_local`
  - Thread
    - Переменная помечена `thread_local`
  - Automatic
    - Переменная определена в block scope и не помечена
    - Параметры функций
  - Dynamic
    - `new` -- контролируется функциями аллокации и деаллокации
    - Implicitly creation
    - Исключения -- unspecified
- Linkage
  - No linkage
    - Имена определённые в block scope:
      - Не помеченные переменные
      - Локальные классы
      - Остальные имена
    - Всё, для чего явно не указан linkage
  - Internal linkage
    - Имена, определённые в namespace scope:
      - Всё, что помечено `static`
      - Переменные, кроме тех, которые:
        - Помечены `static` или `extern`
        - Определены в модуле (но не в private секции)
      - Члены анонимных union
    - **Любые** имена, определённые в анонимном неймспейсе
  - External
    - Автоматически получают language linkage
    - Имена, определённые в namespace scope (но не анонимном namespace):
      - Не-`static` функции
      - Не-`static` константные переменные
      - `extern`
      - enum
      - Классы, и всё, что с ними связано (в том числе, `friend` функции)
    - Имена определённые в block scope:
      - `extern` переменные
      - Функции
  - Module
    - Всё, что определено в модуле и не экспортировано
- Visibility
  - Не определено в стандарте
  - Контролирует, какие символы попадут в какие секции ELF
  - Контролируется compiler-specific атрибутами

# Language linkage
- По стандарту поддерживается только "C++" и "C"
- "C"
  - Если требования к функции не выполнены, молча возвращаемся к "C++"
    - Но параметры остаются "C"
  - На самом деле, не совсем всё так -- см. пример на слайде

# Шаблоны
- Нужно гарантировать, что использованные подстановки появляются ровно один раз, а
  неиспользованные -- не появляются совсем. Традиционная модель с отдельными компилятором и
  линкером так не умеет
  - Две модели, отличия на слайде
